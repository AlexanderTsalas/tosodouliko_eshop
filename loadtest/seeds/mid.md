# `mid` seed

Realistic mid-scale dataset for the Phase B mixed-workload scenarios. Larger and more structured than `thin`/`contention`, with the side data (attributes, specs, custom fields, related products) that real production pages render.

## What this produces

| Table | Rows | Notes |
|---|---|---|
| `categories` | 5 | 3 top-level + 2 child (`epitrapezia` under `paixnidia`, `paramythia` under `vivlia`) — tests hierarchical nav |
| `attributes` | 4 | `color`, `size` (variant axes), `material`, `age` (specifications) |
| `attribute_values` | 20 | 5 colors + 5 sizes + 5 materials + 5 age-ranges (Greek) |
| `products` | 100 | Generated from templates; mix of toys, books, clothing; SKU prefix `MIDxxx` |
| `product_variants` | 1000 | 10 per product = 2 colors × 5 sizes matrix. `attribute_combo = { color: <uuid>, size: <uuid> }` per migration `20260531000002` |
| `inventory_items` | 1000 | Most at 100 units; 10 variants seeded at 1-3 units (for contention scenarios) |
| `product_categories` | ~120 | Every product in its primary category, ~20% also in a child category |
| `product_specifications` | ~180 | Material spec for all 100 products; Age spec for ~80 |
| `custom_fields` | 1 | `gift_message` (text, optional, admin-editable until dispatch) |
| `custom_field_bindings` | ~20 | Gift-message field bound to every 5th product |
| `related_products_associations` | 2 | Both manual-strategy |
| `related_products_manual_picks` | 10 | 5 picks per association |

## Deferred (follow-up seeds)

The following are present in production but **not** in `mid.mjs`. Add them when scenarios need them:
- **Offers + rules + codes** — separate seed file `mid-offers.mjs` (TBD). Schema is complex (rules, conditions, action_*, codes), and the first browse/checkout scenarios don't need them.
- **Customers** — anonymous signin during scenarios creates them. No pre-seed needed.
- **Carriers / fee rules** — migrations seed the built-in carriers + a default COD fee rule. Sufficient for COD checkout.

## What's exposed in the target file

`loadtest/lib/mid-target.json` (generated by this seed, gitignored) contains:
- `counts` — entity totals for the scenario to sanity-check
- `category_slugs` — for navigating to category pages
- `sample_product_slugs` — 20 slugs the scenario can pick from for product detail hits
- `low_stock_variants` — variants seeded at 1-3 units, target list for contention-mix scenarios

## Run

```bash
npx supabase db reset --local
npm run seed:mid
```

Total run time: ~10 seconds (one round-trip per phase, chunked at 100 rows where needed).

## Trigger considerations honored

- `on_variant_inventory_change` (auto-creates inventory_items at 0,0): seed UPSERTs over those rows.
- `trg_validate_attribute_combo` (each combo entry must reference real attribute + value): combos use real attribute_values UUIDs.
- `trg_validate_variant_matrix_shape` (all variants of one product share axes): every product is consistently color+size.

## Why deterministic

All randomness is index-based modular arithmetic, not `Math.random()`. The same seed run twice on a fresh DB produces identical results — important for cross-run comparisons.
